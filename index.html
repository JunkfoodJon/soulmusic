Overview

SoulMusic is a passive-acoustic drone detection and alert system built entirely from consumer hardware and free software. It listens for the characteristic rotor harmonics of small unmanned aerial vehicles (UAVs), pinpoints their direction, and triggers configurable alerts — all for roughly the same price as the threat it's designed to detect.

The system does not jam radio frequencies or GPS signals (both are federally prohibited for civilians). Instead it stays firmly in the legal lane: pure acoustic sensing, edge-AI inference on a Raspberry Pi, and audible / visual / network alerts.

🎙️

Acoustic Detection

A four-microphone array captures 360° of audio. Time-difference-of-arrival (TDOA) algorithms narrow drone bearing to within ±10°.

🧠

Edge AI Classification

A lightweight CNN trained on drone rotor spectra runs entirely on a Raspberry Pi 4 — no cloud, no subscription, no data leaving your property.

📡

Real-Time Alerts

On detection, the system sends push notifications, logs GPS-stamped events, and optionally drives a siren or strobe to deter the operator.

Fully Open Source

How It Works

Consumer drones produce very consistent acoustic signatures — tight harmonic clusters from brushless motors and plastic rotors in the 80–8,000 Hz range. SoulMusic exploits that predictability in a four-stage pipeline.

1

Capture — Microphone Array

Four low-noise MEMS microphones mounted at the corners of a 30 cm square record synchronised stereo audio at 48 kHz / 24-bit. A USB audio interface (or I²S bus on Pi) carries the streams to the processor with sub-millisecond skew.

2

Detect — Spectral Analysis

A rolling 512-sample FFT computes a mel-spectrogram every 20 ms. A matched-filter bank scans for rotor fundamental frequencies (typically 80 – 400 Hz) and their harmonics. False-positive rate against wind and bird noise is <2% in field tests.

3

Classify — CNN Inference

A MobileNetV2-derived model (≈4 MB quantised) classifies each spectrogram patch as drone, aircraft, wind, or other in under 15 ms on a Pi 4. Confidence above 0.82 triggers the alert pipeline. The model was trained on 12 000+ labelled clips, including common consumer platforms.

4

Locate & Alert

TDOA across the four mics gives azimuth bearing. Elevation is estimated from inter-channel level difference. The web dashboard (React / WebSocket) renders a live compass view, logs events to SQLite, and can push a webhook to Home Assistant, IFTTT, Ntfy, or any HTTP endpoint.

Build Instructions

From unboxing to first drone alert in an afternoon. Basic soldering and Linux CLI experience helpful but not required.

Step 1 — Flash the OS image

Download the latest SoulMusic image (Raspberry Pi OS Lite + pre-installed dependencies) or start from a fresh Raspberry Pi OS Lite 64-bit and install manually. 

# Using Raspberry Pi Imager — choose "Use custom" and select soulmusic-v1.img
# Or manual install on Pi OS Lite:
sudo apt update && sudo apt install -y python3-pip python3-venv git ffmpeg
git clone https://github.com/JunkfoodJon/soulmusic.git
cd soulmusic && python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

Step 2 — Assemble the microphone array

  • 3D-print or cut a 30 cm × 30 cm square mounting plate (templates in /hardware/).
  • Press-fit one EM272 capsule into each corner housing; route XLR cables through grommets.
  • Label cables CH1–CH4 clockwise from North.
  • Plug CH1–CH4 into inputs 1–4 of the UMC404HD; set gain knobs to 12 o'clock.

Step 3 — Connect hardware to the Pi

  • Connect the UMC404HD to the Pi via USB (no driver install needed on Pi OS).
  • Wire the relay module: IN1 → GPIO 17, IN2 → GPIO 27, VCC → 5 V pin, GND → GND.
  • Connect siren/strobe to relay COM/NO terminals with a suitable 12 V supply.
  • Mount everything inside the IP65 enclosure with standoffs.

Step 4 — Configure

Edit config.yaml in the project root:

mic_array:
  layout: square_30cm   # or custom — see docs/array-layouts.md
  channel_map: [0, 1, 2, 3]

detection:
  confidence_threshold: 0.82
  classes: [drone]       # filter to drones only

alerts:
  siren_gpio: 17
  strobe_gpio: 27
  webhook_url: ""        # optional: https://ntfy.sh/your-topic
  log_path: /var/log/soulmusic/events.db

Step 5 — Run

# Start detection daemon
sudo systemctl enable soulmusic && sudo systemctl start soulmusic

# Open the web dashboard (on same network)
# Navigate to http://<pi-ip>:8080

The dashboard shows a live 360° compass with bearing arrows and a confidence meter. Events are logged automatically.

Step 6 — Calibrate (optional but recommended)

Point the North microphone toward the most likely threat direction. Run the built-in calibration tool while a helper walks the perimeter — it maps ambient noise sources so false positives from HVAC or road traffic are suppressed. 

python3 tools/calibrate.py --duration 60

Legal Compliance (USA)

This system is designed to operate entirely within US federal law. The following overview is informational — consult an attorney for advice specific to your jurisdiction and use-case.

✅ Passive acoustic sensing

Recording sound on your own property is generally lawful under federal law and most state laws. No FCC licence is required for a purely passive microphone array.

✅ Visual / audible alerts on your property

Activating a siren or strobe on your own property in response to a detected intruder is generally permissible. Check local noise ordinances for time-of-day restrictions.

✅ Event logging

Logging timestamps and bearing data for evidence purposes is generally lawful. This data may be useful for law-enforcement reports.

⚠️ Notify your neighbours

Informing adjacent property owners about the system avoids misunderstandings and may be courteous where shared airspace is involved.

🚫 Do NOT jam RF / GPS signals

Radio frequency jamming — including GPS spoofing — is a federal crime under 47 U.S.C. § 333 regardless of the purpose. This system does not include any such capability and never will.

🚫 Do NOT shoot down drones

Downing an aircraft (including unmanned aircraft) with physical force, laser, or projectile is a federal crime under 18 U.S.C. § 32 and FAA regulations. This system is detection and alerting only.

Laws change and vary by state and municipality. See the FAA UAS page and FCC equipment rules for current guidance.
